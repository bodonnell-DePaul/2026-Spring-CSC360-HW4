# Homework 3: Full-Stack CRUD Application

**CSC 436 — Web Applications (AI-Assisted Development)**
**Due: Before Week 5 class**

---

## Overview

Build a **full-stack CRUD application** using React and ASP.NET Core, and fix a broken app. Use AI tools as coding assistants — but you must understand every line you submit.

---

## Part 1: Billboard Hot 100 CRUD App (80 pts)

A pipe-delimited data file is provided: `billboard-hot-100-no1s.txt`. It contains every song that has reached #1 on the Billboard Hot 100 chart from 1958 to present.

Build a full-stack application using an **ASP.NET Core** API and a **React** front-end. Your API should read and parse this data file on startup, and your app should support full CRUD operations on the data. Store everything in-memory — no database required.

Your app should also include a page where users can browse the full song list and drag-and-drop their favorites into a personal "Top 10" ranked list.

Your app should feel complete and handle real-world usage gracefully.  Use best practice naming conventions when designing your API's.

---

## Part 2: Fix the Broken App (20 pts)

A broken task manager app is provided in `broken-app/`. Run it, find the bugs, fix them, and document each fix in `bug-fixes.md`. For each bug, explain where it was, what was wrong, why, and how you fixed it.

---

## What to Submit

1. Your CRUD app source code
2. `bug-fixes.md` with your documented fixes
3. `ai-log.md` — your AI usage log for both parts, including examples of modifying or rejecting AI suggestions
4. Screenshots of your working CRUD app
